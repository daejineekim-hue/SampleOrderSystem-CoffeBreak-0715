# 출고 처리 (Shipment)

## Overview

출고(Shipment)는 재고가 충분해져 승인(CONFIRMED)된 주문을 실제로 내보내는 마지막 처리 단계이다.
재고가 충분하지 않아 대기 중이던 주문이 생산(Production)을 거쳐 재고를 확보하면 상태가 `CONFIRMED`로 전환되고,
이 `CONFIRMED` 상태의 주문에 대해서만 출고를 실행할 수 있다. 출고가 실행되면 주문 상태는 `RELEASE`로 전환되며,
이는 해당 주문의 처리가 완료되었음을 의미한다.

본 문서는 이후 GoogleTest 테스트 케이스로 변환되는 것을 전제로 하므로, 모든 동작은 Given-When-Then 형식으로 구체적으로 기술한다.

**주문 상태(Order Status) 전이 흐름 (참고)**: `RESERVED`(재고 부족, 생산 대기) → `PRODUCING`(생산 중) →
`CONFIRMED`(재고 확보 완료, 출고 대기) → `RELEASE`(출고 완료). 그 외 `REJECTED`(승인/생산 거부) 상태가 존재할 수 있다.
상태 전이의 세부 규칙은 각각 주문 승인(order-approval.md) 및 생산(production-line.md) 문서에서 정의하며,
본 문서는 `CONFIRMED → RELEASE` 전이만을 다룬다.

---

## Precondition

출고를 실행하려는 주문은 반드시 `CONFIRMED` 상태여야 한다.

- **[전제 조건] 출고 대상 주문은 `CONFIRMED` 상태여야 한다.**
  - Given 주문이 존재하고 그 상태가 `CONFIRMED`일 때
  - When 해당 주문에 대해 출고를 실행하면
  - Then 출고 처리가 진행된다 (아래 "출고 실행 Acceptance Criteria" 참조).

- **[전제 조건 위반 - RESERVED] 재고 확보 전(RESERVED) 주문은 출고할 수 없다.**
  - Given 주문 상태가 `RESERVED`(재고 부족으로 생산 대기 중)일 때
  - When 해당 주문에 대해 출고를 실행하면
  - Then 출고는 실패하고(예외 발생 또는 실패 반환값), 주문 상태는 `RESERVED`로 유지된다.

- **[전제 조건 위반 - PRODUCING] 생산 중(PRODUCING)인 주문은 출고할 수 없다.**
  - Given 주문 상태가 `PRODUCING`일 때
  - When 해당 주문에 대해 출고를 실행하면
  - Then 출고는 실패하고, 주문 상태는 `PRODUCING`으로 유지된다.

- **[전제 조건 위반 - 이미 RELEASE] 이미 출고 완료된 주문은 다시 출고할 수 없다.**
  - Given 주문 상태가 이미 `RELEASE`일 때
  - When 해당 주문에 대해 출고를 다시 실행하면
  - Then 출고는 실패하고, 주문 상태는 `RELEASE`로 유지된다 (중복 출고로 인한 상태 변경이나 재고 변동이 발생하지 않는다).

- **[전제 조건 위반 - REJECTED] 거부(REJECTED)된 주문은 출고할 수 없다.**
  - Given 주문 상태가 `REJECTED`일 때
  - When 해당 주문에 대해 출고를 실행하면
  - Then 출고는 실패하고, 주문 상태는 `REJECTED`로 유지된다.

- **[전제 조건 위반 - 존재하지 않는 주문] 존재하지 않는 주문 ID로 출고를 요청하면 실패한다.**
  - Given 시스템에 등록되지 않은 주문 ID `"O999"`가 있을 때
  - When `"O999"`에 대해 출고를 실행하면
  - Then 출고는 실패한다(예외 발생 또는 실패 반환값).

- **[명확한 오류] `CONFIRMED`가 아닌 상태에서의 출고 실패는 원인을 특정할 수 있는 오류로 처리되어야 한다.**
  - Given 주문 상태가 `CONFIRMED`가 아닌 임의의 상태(`RESERVED`, `PRODUCING`, `RELEASE`, `REJECTED`)일 때
  - When 해당 주문에 대해 출고를 실행하면
  - Then 실패 사유(예: "현재 주문 상태는 CONFIRMED가 아님")를 확인할 수 있는 오류/예외 메시지 또는 실패 코드가 반환된다.
    (단순히 아무 동작도 하지 않고 무시되어서는 안 된다.)

## Operations

### 1. 출고 실행 (Execute Shipment)

특정 주문 ID를 지정하여 출고를 실행한다. 대상 주문은 `CONFIRMED` 상태여야 하며, 처리에 성공하면 주문 상태가
`RELEASE`로 전환된다.

#### Acceptance Criteria

- **[정상 출고 - 상태 전이] `CONFIRMED` 상태의 주문은 출고 실행 시 `RELEASE`로 전환된다.**
  - Given 주문 `"O001"`의 상태가 `CONFIRMED`이고 수량이 `50`일 때
  - When `"O001"`에 대해 출고를 실행하면
  - Then 출고는 성공하고, `"O001"`의 상태는 `RELEASE`로 전환된다.

- **[정상 출고 - 반환값/결과] 출고 성공 시 처리 결과를 확인할 수 있다.**
  - Given 주문 `"O001"`의 상태가 `CONFIRMED`일 때
  - When `"O001"`에 대해 출고를 실행하면
  - Then 성공을 나타내는 반환값(또는 예외 없음)이 반환되고, 주문 조회 시 상태가 `RELEASE`로 확인된다.

- **[다건 처리 독립성] 한 주문의 출고가 다른 주문에 영향을 주지 않는다.**
  - Given 주문 `"O001"`, `"O002"`가 모두 `CONFIRMED` 상태일 때
  - When `"O001"`에 대해서만 출고를 실행하면
  - Then `"O001"`의 상태는 `RELEASE`로 전환되고, `"O002"`의 상태는 `CONFIRMED`로 그대로 유지된다.

- **[설계 노트 - 재고 재차감 없음] 출고 실행 시점에는 재고를 추가로 차감하지 않는다.**
  - Given 시료 `"S001"`의 재고가 승인/생산 단계에서 이미 주문 수량만큼 차감되어 `stock=50`이고,
    주문 `"O001"`(시료 `"S001"`, 수량 `30`)의 상태가 `CONFIRMED`일 때
  - When `"O001"`에 대해 출고를 실행하면
  - Then 출고는 성공하여 상태가 `RELEASE`로 전환되지만, 시료 `"S001"`의 재고는 출고 실행 전후로 동일하게
    `stock=50`으로 유지된다 (재고는 주문 승인(order-approval.md) 또는 생산 완료(production-line.md)
    시점에 이미 차감되었으므로, 출고 단계에서 재차감하면 이중 차감(double-decrement) 오류가 발생한다.
    따라서 출고 처리는 상태 전이(`CONFIRMED → RELEASE`)만 수행하고 재고에는 관여하지 않는 것으로 설계한다).

### 2. 출고 가능 목록 조회 (List Shippable Orders)

현재 `CONFIRMED` 상태인, 즉 출고가 가능한 주문 목록만을 조회한다.

#### Acceptance Criteria

- **[정상 조회 - CONFIRMED만 필터링] 여러 상태의 주문이 섞여 있어도 `CONFIRMED` 상태만 반환한다.**
  - Given 주문 `"O001"`(상태 `CONFIRMED`), `"O002"`(상태 `RESERVED`), `"O003"`(상태 `PRODUCING`),
    `"O004"`(상태 `RELEASE`), `"O005"`(상태 `REJECTED`), `"O006"`(상태 `CONFIRMED`)가 등록되어 있을 때
  - When 출고 가능 목록을 조회하면
  - Then 결과에는 `"O001"`과 `"O006"`만 포함되고, 나머지 주문은 포함되지 않는다.

- **[빈 목록] `CONFIRMED` 상태의 주문이 하나도 없으면 빈 목록을 반환한다.**
  - Given 등록된 주문이 모두 `RESERVED`, `PRODUCING`, `RELEASE`, `REJECTED` 상태 중 하나이고 `CONFIRMED`인
    주문이 없을 때
  - When 출고 가능 목록을 조회하면
  - Then 빈 목록(size 0)이 반환된다. (예외를 던지지 않는다)

- **[빈 목록 - 주문 없음] 등록된 주문 자체가 없으면 빈 목록을 반환한다.**
  - Given 시스템에 등록된 주문이 하나도 없을 때
  - When 출고 가능 목록을 조회하면
  - Then 빈 목록(size 0)이 반환된다.

- **[출고 후 목록에서 제외] 출고가 완료된 주문은 이후 조회에서 제외된다.**
  - Given 주문 `"O001"`, `"O002"`가 모두 `CONFIRMED` 상태이고, `"O001"`에 대해 출고를 실행하여
    상태가 `RELEASE`로 전환되었을 때
  - When 다시 출고 가능 목록을 조회하면
  - Then 결과에는 `"O002"`만 포함되고 `"O001"`은 포함되지 않는다.

## Edge Cases

- **[중복 출고 실패] 같은 주문에 대해 출고를 두 번 실행하면 두 번째 시도는 실패한다.**
  - Given 주문 `"O001"`의 상태가 `CONFIRMED`이고, 첫 번째 출고 실행이 성공하여 상태가 `RELEASE`로
    전환되었을 때
  - When 동일한 `"O001"`에 대해 다시 출고를 실행하면
  - Then 두 번째 출고 시도는 실패한다(더 이상 `CONFIRMED` 상태가 아니므로). 주문 상태는 `RELEASE`로
    유지되며, 재고나 상태에 추가 변경이 발생하지 않는다.

- **[출고 실패 후 재시도 불가] 전제 조건 위반으로 실패한 출고는 상태를 변경하지 않으므로, 원인이
  해소되기 전까지 재시도해도 계속 실패한다.**
  - Given 주문 `"O001"`의 상태가 `RESERVED`이고, 출고 실행이 실패했을 때
  - When 재고 확보나 생산 완료 없이 `"O001"`에 대해 다시 출고를 실행하면
  - Then 여전히 실패하고 상태는 `RESERVED`로 유지된다. (상태가 `CONFIRMED`로 전환된 이후에만 출고가
    성공할 수 있다.)

- **[수량 0 이하 주문] 수량이 0 이하인 주문은 애초에 생성/승인 단계에서 차단되어야 하므로, 출고 단계에서는
  다루지 않는다.**
  - 주문 수량의 유효성 검증은 주문 생성 및 승인(order-approval.md) 문서의 범위이며, 출고 단계에서는
    이미 유효한 수량을 가진 `CONFIRMED` 주문만을 대상으로 한다고 가정한다.

## Non-Goals

- **재고 차감/증가 로직**: 출고 처리 자체는 재고에 관여하지 않는다. 재고 차감은 주문 승인
  (order-approval.md) 또는 생산 완료(production-line.md) 시점에 이미 수행된 것으로 전제한다.
- **주문 승인/생산 로직**: `RESERVED → PRODUCING → CONFIRMED` 전이의 세부 규칙(재고 확인, 생산 완료 판정 등)은
  본 문서의 범위가 아니며 각각 order-approval.md, production-line.md에서 다룬다.
- **영속성(Persistence)**: 출고 이력이나 주문 상태를 파일로 저장/복원하는 방식은 본 문서에서 다루지 않는다.
- **출고 취소/반품**: 한 번 `RELEASE`로 전환된 주문을 다시 이전 상태로 되돌리는 취소/반품 기능은 과제
  명세에 명시되지 않았으므로 본 문서의 범위 밖으로 간주한다.
- **동시성(Concurrency)**: 멀티스레드 환경에서 동일 주문에 대한 동시 출고 요청 처리는 본 콘솔 애플리케이션의
  범위 밖으로 간주한다.
