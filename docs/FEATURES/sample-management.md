# 시료 관리 (Sample Management)

## Overview

시료(Sample)는 반도체 시료 생산주문관리 시스템의 가장 기본 단위이다. 각 시료는 고유한 이름과 속성(생산시간, 수율 등)을 가지며,
시스템에 **등록된 시료만 주문(Order) 가능**하다. 본 문서는 시료 엔티티의 등록/조회/검색 기능에 대한 비즈니스 규칙과
테스트 가능한 인수 기준(Acceptance Criteria)을 정의한다.

**수율(Yield) 정의**: `수율 = 정상적인 시료 수 / 총 생산 시료 수`
예) 100개 생산 중 정상 물품 90개 → 수율 = 0.9

이 문서는 이후 GoogleTest 테스트 케이스로 변환되는 것을 전제로 하므로, 모든 동작은 Given-When-Then 형식으로 구체적으로 기술한다.

## Data Model

### `Sample`

| 필드 | 타입 | 설명 | 검증 규칙 |
|---|---|---|---|
| `id` | `std::string` | 시료 고유 식별자 | 공백이 아닌 문자열이어야 하며, 시스템 내에서 유일(unique)해야 함 |
| `name` | `std::string` | 시료 이름 | 공백(빈 문자열) 불가 |
| `avgProcessTimeMin` | `double` (또는 `int`, 분 단위) | 평균 생산시간(분) | `avgProcessTimeMin > 0` |
| `yieldRate` | `double` | 수율 | `0 < yieldRate <= 1` (0은 불가, 1은 허용 — 전량 정상 생산) |
| `stock` | `int` | 현재 재고 수량 | `stock >= 0` |

#### 설계 결정 (Design Decisions)

- **초기 재고(`stock`) 필드 추가**: 원본 과제 명세에는 시료 등록 시 필요한 속성으로 "시료 ID, 이름, 평균 생산시간, 수율"만 명시되어 있으나,
  "시료 조회 시 현재 재고 수량도 함께 표시"해야 하는 요구사항을 만족하려면 재고 데이터가 반드시 필요하다.
  따라서 시료 등록 시 `stock`을 **0 이상의 정수**로 함께 입력받는 것으로 설계한다 (미입력 시 기본값 0).
- **검색 방식**: `시료 검색`은 이름 등 속성에 대해 **부분 문자열 매칭(substring match)** 으로 설계한다. 정확히 일치하는 항목만
  찾는 것이 아니라, 검색어를 포함하는 모든 시료를 반환한다.
- **검색 대소문자 구분**: 부분 문자열 매칭은 **대소문자를 구분(case-sensitive)** 한다. (예: "abc" 검색 시 "ABC123"은 매칭되지 않음)
- **ID 발급 방식**: 시료 ID는 사용자가 등록 시 직접 지정하는 문자열로 가정한다 (자동 채번은 본 기능의 범위 밖).
- **영속성(파일 저장 등)** 은 이 문서에서 다루지 않으며 별도 문서(persistence 스펙)에서 정의한다.

## Operations

### 1. 시료 등록 (Register Sample)

새로운 시료를 시스템에 추가한다. 입력값: `id`, `name`, `avgProcessTimeMin`, `yieldRate`, (`stock`, 선택 — 기본값 0).

#### Acceptance Criteria

- **[정상 등록] 유효한 값으로 등록하면 성공한다.**
  - Given 시스템에 아직 등록되지 않은 시료 ID `"S001"`이 있을 때
  - When `id="S001"`, `name="Wafer-A"`, `avgProcessTimeMin=30`, `yieldRate=0.9`, `stock=100`으로 등록을 요청하면
  - Then 등록은 성공하고, 시료 목록 조회 시 해당 시료가 포함된다.

- **[정상 등록 - stock 생략] stock을 지정하지 않으면 0으로 초기화된다.**
  - Given 유효한 `id`, `name`, `avgProcessTimeMin`, `yieldRate`가 주어졌을 때
  - When `stock` 값 없이 등록을 요청하면
  - Then 등록은 성공하고, 해당 시료의 `stock`은 `0`이다.

- **[중복 ID 거부] 이미 존재하는 ID로 등록하면 실패한다.**
  - Given 시스템에 `id="S001"`인 시료가 이미 등록되어 있을 때
  - When 동일한 `id="S001"`로 다른 이름/속성을 가진 시료 등록을 요청하면
  - Then 등록은 실패하고(예외 발생 또는 실패 반환값), 기존 시료 정보는 변경되지 않는다.

- **[빈 이름 거부] 이름이 빈 문자열이면 등록이 실패한다.**
  - Given 유효한 `id`, `avgProcessTimeMin`, `yieldRate`가 주어졌을 때
  - When `name=""`으로 등록을 요청하면
  - Then 등록은 실패한다.

- **[빈 ID 거부] ID가 빈 문자열이면 등록이 실패한다.**
  - Given 유효한 `name`, `avgProcessTimeMin`, `yieldRate`가 주어졌을 때
  - When `id=""`으로 등록을 요청하면
  - Then 등록은 실패한다.

- **[유효하지 않은 수율 거부 - 0] 수율이 0이면 등록이 실패한다.**
  - Given 유효한 `id`, `name`, `avgProcessTimeMin`이 주어졌을 때
  - When `yieldRate=0`으로 등록을 요청하면
  - Then 등록은 실패한다.

- **[유효하지 않은 수율 거부 - 음수] 수율이 음수이면 등록이 실패한다.**
  - When `yieldRate=-0.1`로 등록을 요청하면
  - Then 등록은 실패한다.

- **[유효하지 않은 수율 거부 - 1 초과] 수율이 1을 초과하면 등록이 실패한다.**
  - When `yieldRate=1.1`로 등록을 요청하면
  - Then 등록은 실패한다.

- **[수율 경계값 1 허용] 수율이 정확히 1이면 등록이 성공한다.**
  - When `yieldRate=1.0`으로 등록을 요청하면
  - Then 등록은 성공한다.

- **[유효하지 않은 평균 생산시간 거부 - 0] 평균 생산시간이 0이면 등록이 실패한다.**
  - When `avgProcessTimeMin=0`으로 등록을 요청하면
  - Then 등록은 실패한다.

- **[유효하지 않은 평균 생산시간 거부 - 음수] 평균 생산시간이 음수이면 등록이 실패한다.**
  - When `avgProcessTimeMin=-10`으로 등록을 요청하면
  - Then 등록은 실패한다.

- **[유효하지 않은 재고 거부 - 음수] 재고가 음수이면 등록이 실패한다.**
  - When `stock=-1`로 등록을 요청하면
  - Then 등록은 실패한다.

- **[재고 경계값 0 허용] 재고가 정확히 0이면 등록이 성공한다.**
  - When `stock=0`으로 등록을 요청하면
  - Then 등록은 성공한다.

### 2. 시료 조회 (List Samples)

등록된 모든 시료 목록을 확인한다. 각 시료의 현재 재고 수량도 함께 표시되어야 한다.

#### Acceptance Criteria

- **[전체 목록 조회] 등록된 시료가 있으면 모두 반환한다.**
  - Given 시료 `"S001"`, `"S002"`가 등록되어 있을 때
  - When 시료 목록을 조회하면
  - Then 두 시료 모두 결과에 포함되고, 각 결과에는 `id`, `name`, `avgProcessTimeMin`, `yieldRate`, `stock`이 포함된다.

- **[빈 목록] 등록된 시료가 없으면 빈 목록을 반환한다.**
  - Given 시스템에 등록된 시료가 하나도 없을 때
  - When 시료 목록을 조회하면
  - Then 빈 목록(size 0)이 반환된다. (예외를 던지지 않는다)

- **[재고 반영] 조회 결과에는 최신 재고 수량이 포함된다.**
  - Given `stock=50`으로 등록된 시료 `"S001"`이 있고, 이후 재고가 `30`으로 변경되었을 때
  - When 시료 목록을 조회하면
  - Then `"S001"`의 `stock`은 `30`으로 조회된다.

### 3. 시료 검색 (Search Samples)

이름 등 속성으로 특정 시료를 검색한다. **부분 문자열 매칭(substring match)** 으로 동작하며, **대소문자를 구분**한다.

#### Acceptance Criteria

- **[정확 일치 검색] 검색어가 이름과 완전히 일치하면 해당 시료를 반환한다.**
  - Given `name="Wafer-A"`인 시료가 등록되어 있을 때
  - When `"Wafer-A"`로 검색하면
  - Then 해당 시료가 결과에 포함된다.

- **[부분 문자열 검색] 검색어가 이름의 일부만 포함해도 매칭된다.**
  - Given `name="Wafer-A"`인 시료가 등록되어 있을 때
  - When `"wafer"`가 아닌 `"afer"`로 검색하면 (부분 문자열)
  - Then 해당 시료가 결과에 포함된다.

- **[다중 매칭] 여러 시료 이름이 검색어를 포함하면 모두 반환한다.**
  - Given `name="Wafer-A"`, `name="Wafer-B"`, `name="Chip-C"`인 시료가 등록되어 있을 때
  - When `"Wafer"`로 검색하면
  - Then `"Wafer-A"`, `"Wafer-B"` 두 시료가 결과에 포함되고, `"Chip-C"`는 포함되지 않는다.

- **[대소문자 구분 - 결정 사항] 검색은 대소문자를 구분하므로 대소문자가 다르면 매칭되지 않는다.**
  - Given `name="Wafer-A"`인 시료가 등록되어 있을 때
  - When `"WAFER"`(대문자)로 검색하면
  - Then 결과는 빈 목록이다. (대소문자 무시 매칭을 원할 경우 향후 별도 옵션으로 확장, 본 기능 범위 아님)

- **[매칭 없음] 검색어와 일치하는 시료가 없으면 빈 목록을 반환한다.**
  - Given 시스템에 `"Wafer-A"`만 등록되어 있을 때
  - When `"NonExistent"`로 검색하면
  - Then 빈 목록(size 0)이 반환된다. (예외를 던지지 않는다)

- **[빈 검색어] 검색어가 빈 문자열이면 등록된 모든 시료를 반환한다.**
  - Given 시료 `"S001"`, `"S002"`가 등록되어 있을 때
  - When 검색어 `""`로 검색하면
  - Then 모든 시료가 결과에 포함된다. (빈 문자열은 모든 문자열의 부분 문자열이므로)

- **[ID로 검색] ID를 검색어로 사용해도 부분 문자열 매칭이 적용된다.**
  - Given `id="S001"`인 시료가 등록되어 있을 때
  - When `"S00"`으로 검색하면
  - Then 해당 시료가 결과에 포함된다.

## Non-Goals

- **영속성(Persistence)**: 시료 데이터를 파일(텍스트/CSV/바이너리 등)로 저장하고 불러오는 방식, 저장 포맷, 파일 I/O 오류 처리 등은
  본 문서의 범위가 아니며 별도의 persistence 관련 문서에서 다룬다.
- **주문(Order) 처리 로직**: 시료를 이용한 주문 생성, 주문 시 재고 차감/검증 등 주문 도메인의 세부 규칙은 별도의
  주문 관리(Order Management) 문서에서 다룬다. 본 문서는 "등록된 시료만 주문 가능하다"는 전제 조건만 언급하며,
  주문 자체의 동작은 다루지 않는다.
- **동시성(Concurrency)**: 멀티스레드 환경에서의 동시 등록/조회에 대한 처리는 본 콘솔 애플리케이션의 범위 밖으로 간주한다.
- **수정/삭제(Update/Delete)**: 과제 명세에 명시되지 않은 시료 정보 수정 및 삭제 기능은 본 문서에서 다루지 않는다.
